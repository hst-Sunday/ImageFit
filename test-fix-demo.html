🎯 图像缩放问题修复演示

❌ 修复前的问题

当同时提供 width 和 height 参数时：

// 用户期望：1600x1600 → 1280x800 POST /api/process FormData: { width: 1280, height: 800, image: file } // 实际结果：800x800 (不是期望的 1280x800) // 原因：使用了 fit: 'inside' 模式保持宽高比

问题：使用 fit: 'inside' 模式会保持原图宽高比，导致输出尺寸不是用户期望的精确尺寸。

✅ 修复后的解决方案

现在默认使用 fit: 'fill' 模式：

// 现在的行为：1600x1600 → 1280x800 POST /api/process FormData: { width: 1280, height: 800, image: file } // 实际结果：1280x800 (精确的目标尺寸！) // 原因：默认使用 fit: 'fill' 模式强制精确尺寸

解决方案：

  • 默认使用 fit: 'fill' 确保精确尺寸
  • 新增 fit 参数支持多种缩放模式
  • 保持向后兼容性

📋 新的缩放模式说明

模式 行为 适用场景
fill (默认) 强制调整到精确尺寸，可能轻微变形 需要精确尺寸，允许轻微变形
cover 保持宽高比，覆盖整个目标区域，可能裁剪 填满区域，不介意裁剪
contain 保持宽高比，完全适应目标尺寸，可能有空白 保持完整图像，允许空白

🔧 API 使用示例

# 默认行为 (fit=fill) curl -X POST http://localhost:8000/api/process \ -F "image=@photo.jpg" \ -F "width=1280" \ -F "height=800" # 显式指定缩放模式 curl -X POST http://localhost:8000/api/process \ -F "image=@photo.jpg" \ -F "width=1280" \ -F "height=800" \ -F "fit=cover" # 仅缩放 (不压缩) curl -X POST http://localhost:8000/api/resize \ -F "image=@photo.jpg" \ -F "width=1280" \ -F "height=800" \ -F "fit=fill"
